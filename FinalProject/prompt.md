Full initial prompt below: 

I want to develop a mobile puzzle game which has features inspired by the mobile game Puzzles & Dragons, by Gungho Online Entertainment. The primary feature I want to implement is the grid-based drag and drop orb matching feature where the user holding and dragging the orb is set on a timer. I especially want the ability for adjacent orbs to swap places with the orb being dragged around, just like in Puzzles & Dragons. I would also like the ability for orbs to swap with orbs diagonally to it if the player drags diagonally, not just adjacent swapping. Once the player lets go of their drag, or the drag timer runs out, the game will assess the board state for all of the orb matches. Orb matches will be made when 3 orbs of the same elements are aligned, either vertically or horizontally. However, matching can also occur if there are additional orbs of the same element that are part of that vertical/horizontal match, that also contributes to the match. 

I want to implement a point system where there is a multiplier applied to matches that have more than 3 orbs matched. So there should be a baseline points scored for 3 orb matches, but for each additional orb that contributes to that match, like a row of 4 matching orbs, the player earn bonus points, and this continues to apply for additional orbs, like 5-orb matches. Additionally, like Puzzles and Dragons, I want to add a combo multiplier for each additional match that occurs once the player finishes dragging. The game should check the board for any possible matches. Once the matches have been identified, they will be removed and will add to the multiplier for the points earned after all matches have been complete. 

However, like Puzzles and Dragons, once the matches from the initial board have resolved and disappeared, orbs that are above where the empty cells that the matched orbs are will drop down and fill in the spot accordingly, and the column will drop down accordingly. As the orbs drop down, random orbs will drop in to fill in whatever empty grid cells remain near the top of the grid. Then the game will continue to assess if there are any orb matches after this phase of refilling the orb completes. If there are, it will continue to add to the combo counter. Then the process of filling it the grid in with a random assortment of orbs and checking for matches will loop until no matches are found and the grid is fully filled in. At this point, the game should calculate and apply the points and multiplier and add it to the current score. After that, the multiplier will reset and the player is allowed to drag orbs again. There will be a timer for the game session, let's set it to 2 minutes for now. While the process of assessing matches and filling in the grid is happening, the timer should not be running. It should after the points have been earned. 

The grid for the board of orbs will be a 5 by 6 (row by column) grid. There will be 5 different orb elements. For now, just make them different colors to specify the different elements. Add animation to the orbs swapping positions during the dragging phase. I also want a trailing animation following as the player drags the orb. As for the layout of the gameplay screen. The board grid should be near the center of the screen. At the top of the screen, there should be the current timer, combo counter, the current score, and the pause button from left to right. If possible, as the combo counter increases, make it briefly change colors and grow slightly in size as an animation, before returning back to normal. This should occur at each combo count increment. If the player presses on the pause button, a new overlay should appear over the current view with the button options to resume, restart, or return to the title screen. Until the player presses one of the buttons in this overlay, the timer should be paused and will resume only when “resume” is pressed. There will be three different pages in this app. First is the title page, the gameplay page, and the game over screen. The title page will be simple for now; it will have a large title labelled “CapyBerry” and a ”start game”button. The game over screen will contain the saved highest score that the player has gotten, the total score from that game session, a “play again” button, and a quit button, which redirects to the title screen.
